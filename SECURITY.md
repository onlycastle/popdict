# Security

Please report security issues privately. Do not open a public issue for suspected
vulnerabilities, exposed credentials, account data, or abuse paths.

## Reporting

Use GitHub Security Advisories on the public repository when available. If advisories
are not enabled, use the contact method listed on PopDict's privacy policy page.

Include:

- A concise description of the issue.
- Steps to reproduce or a proof of concept.
- Affected versions or commit SHAs, if known.
- Whether any credentials or user data may be exposed.

## Secrets

PopDict should never ship private keys, Supabase service role keys, STANDS4 tokens,
Apple signing material, notary credentials, or personal `.env` files. If you believe
a secret was committed, report it privately so it can be revoked and rotated.
