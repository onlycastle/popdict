# Dictionary data

Lookup flows from `src/views/SearchView.tsx` through
[DictionaryService.ts](../../src/services/dictionary/DictionaryService.ts).
Single words use [FreeDictionarySource.ts](../../src/services/dictionary/FreeDictionarySource.ts).
Multi-word input queries it and [KaikkiPhraseSource.ts](../../src/services/dictionary/KaikkiPhraseSource.ts)
concurrently, merging successful results from the public read-only phrase table.

## Sources

Free Dictionary supplies English definitions, examples, phonetics, and
source/license metadata. Recorded audio is used only when its phonetic carries
source and license attribution; otherwise the app uses system TTS.

The old Korean-only dictionary integration was removed 2026-07-03. The DB teardown is
[20260703230000_remove_korean_support.sql](../../supabase/migrations/20260703230000_remove_korean_support.sql);
the earlier Korean-era migrations stay in place because applied migrations
are immutable.

Translations are a separate English→target-language layer available without sign-in.
[TranslationService.ts](../../src/services/TranslationService.ts) reads up to
three ranked rows directly from `word_translations` after a successful
canonical single-word Free Dictionary result. RLS permits `anon` and
`authenticated` reads and denies all client writes.
There is no translation edge function or live translation vendor.

The tracked dataset under [data/translations](../../data/translations/README.md) is
generated by [build-dataset.mjs](../../scripts/translations/build-dataset.mjs)
from the pinned Kaikki English Wiktionary snapshot and all 5,049 valid distinct
normalized NGSL-GR learner headwords. It filters variants, ranks at most three
equivalents per target language, and completes exact-entry gaps with one vetted
alias sense or an explicitly disclosed PopDict-contributor equivalent. The
manifest records both fallback counts. The complete dataset is CC BY-SA 4.0
rather than MIT.

The licensed phrase dataset under [data/phrases](../../data/phrases/README.md)
contains filtered idiomatic, phrase, prepositional-phrase, and proverb senses,
capped at three senses per normalized phrase with source attribution.

## Supabase

- Edge functions include [idioms](../../supabase/functions/idioms/index.ts),
  [downloads](../../supabase/functions/downloads/index.ts), feedback, events,
  and quiz.
  `idioms` is a compatibility tombstone that returns `upstream_disabled` and
  performs no upstream request. Active functions read their
  secrets (service-role key and endpoint tokens) via
  `Deno.env.get` — the `supabase-boundary` gate enforces the boundary
  (learning L-006). Functions using custom token auth deploy with
  `--no-verify-jwt`.
- Migrations: [supabase/migrations/](../../supabase/migrations/) — saved words,
  tags/notes/snapshots, review state, download tracking, quiz data, public
  translation/phrase datasets, and historical create/teardown pairs.

## Saved words

[SavedWordsRepository.ts](../../src/services/SavedWordsRepository.ts) against
the `saved_words` table (anon key via
[supabaseClient.ts](../../src/services/supabaseClient.ts)); UI in
`src/views/SavedWordsView.tsx`. Saved cards persist dictionary snapshots,
translations, tags, private notes, and review state. Legacy visible rows enrich
lazily with concurrency two; filters and CSV export operate on the loaded set.

## Study digest

Weekly quiz emails turn saved words into spaced-repetition exercises. One
edge function owns the whole loop:
[quiz/index.ts](../../supabase/functions/quiz/index.ts).

| Action | Trigger | Does |
|---|---|---|
| `send` | Vercel Cron via [site/app/api/cron/quiz/quiz.ts](../../site/app/api/cron/quiz/quiz.ts) (`x-quiz-token`) | Builds each due user's digest from saved words + Leitner state, emails it through Resend. |
| `answer` | Email link, proxied by [site/app/quiz/answer/route.ts](../../site/app/quiz/answer/route.ts) | Records the choice, advances the Leitner box, updates streak. |
| `review` | Fetched by [site/app/quiz/result/page.tsx](../../site/app/quiz/result/page.tsx) | Returns the study material + outcome to render the full study card. |
| `unsubscribe` | Proxied by [site/app/quiz/unsubscribe/route.ts](../../site/app/quiz/unsubscribe/route.ts) → [unsubscribed page](../../site/app/quiz/unsubscribed/page.tsx) | Flips `quiz_preferences.enabled` off. |

Study cards are built deterministically from saved-word snapshots, using other
eligible saved words as distractors. Existing material validated by
[materials.ts](../../supabase/functions/quiz/materials.ts) is a legacy fallback;
no live model is called. Cached legacy rows live in
[word_study_materials](../../supabase/migrations/20260705120000_create_word_study_materials.sql),
and are not generated by the current runtime. Count and session share one
eligibility function and the same eight-card cap.

In-app answer requests require the saved word owner's JWT and accept only
questions from an `app` quiz. Email answer links remain UUID capabilities and
accept only `email` questions. The atomic database function derives streak
updates from that persisted source, so app reviews cannot affect the weekly
email streak regardless of which HTTP route or legacy flag reaches it.

Spaced repetition (Leitner boxes 1-5) lives in `word_reviews`, created in
[20260703130000_create_quiz_schema.sql](../../supabase/migrations/20260703130000_create_quiz_schema.sql)
alongside `quiz_preferences`/`quizzes`/`quiz_questions`; the box math is in
[lib.ts](../../supabase/functions/quiz/lib.ts).

In the app,
[QuizPreferencesRepository.ts](../../src/services/QuizPreferencesRepository.ts)
reads/writes `quiz_preferences`;
[useSaveWord.ts](../../src/hooks/useSaveWord.ts) prompts opt-in once, on a
user's 5th saved word. Local review reminders remain separate from weekly email.
